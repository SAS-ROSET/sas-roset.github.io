2. Keys for the SAS-ROS Cipher by: saaiqSAS Last Updated: 13/6/2024 SAS-ROS Cipher uses two types of keys: Dynamic Keys and Static Keys. Both keys together form a substitution table for the substitution process, making each key type alone meaningless. Different key combinations produce unique substitution tables. Example 2.1: Substitution table produced by a key pair Dynamic Key: { j, i, d, a, h, c, g, f, e, b } Static Key : { 4, 8, 2, 7, 1, 6, 0, 5, 9, 3 } ___________________ |___m1___|___m2___| | a -> f | a -> b | | b -> a | b -> e | | c -> g | c -> f | | d -> d | d -> d | | e -> b | e -> i | | f -> c | f -> a | | g -> j | g -> c | | h -> i | h -> j | | i -> e | i -> h | | j -> h | j -> g | |_________________| (Notice how 'd' gets substituted to 'd' in both methods. Learn why in the next Section - Laws of the SAS-ROS Cipher) Dynamic Keys Dynamic Keys are generated by randomly shuffling a set of non-identical objects, usually of the same type (e.g., characters, bytes). With a secure entropy source, the keyspace for the Dynamic Key (D-Space) includes all possible permutations of the objects in the set, given by the factorial of the set's length (L!). Static Keys Static Keys are generated by randomly shuffling a set of orderly indexes up to the length of the Dynamic Key. With a secure entropy source, the keyspace for the Static Key (S-Space) includes all possible permutations of the indexes in the set, also given by the factorial of the set's length (L!). Total Effective Keyspace While D-Space and S-Space represent the keyspaces for Dynamic Keys and Static Keys, the total keyspace (KC-Space) includes all possible combinations of these keys. This is given by the product of D-Space and S-Space, or the square of the factorial of the key length (L!)^2.

Although KC-Space ((L!)^2) is the total keyspace for the ROS Cipher, it is not considered the effective keyspace since it can be simplified. The maximum substitution tables for a given length are given by L!. In Example 2.1, the column of any side of the arrows in any method represents a permutation from the D-Space. Thus, the total substitution tables producable (Sub-Space) are equal the length of D-Space - L!. Therefore, the effective keyspace for the ROS Cipher (eKC-Space) is L!.

KC-Space being greater than Sub-Space indicates that not all key combinations from KC-Space produce unique substitution tables. Some combinations produce identical tables, leading to identical substitutions. More details on this, including the frequency of substitution table collisions, will be discussed in the next section - Laws of the SAS-ROS Cipher.
